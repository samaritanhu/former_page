In the third week, we talked about hard problems, digital signatures, one-way functions, ElGamal public-key encryption and HTTPS/SSL protocol.

Digital Signature 数字签名

1) Gen: A randomized algorithm which outputs VK and SK

2) Sign: A (possibly randomized) algorithm. Takes input SK and M. Outputs σ.

3) Verify: Takes input (M, σ, VK). Outputs 0/1.

Correctness: If (VK, SK) are output of Gen, must have Verify(M, Sign(M, SK), VK) = 1

Homework 2

Q1: Consider the one-time digital signature scheme discussed in the lecture. Suppose the adversary can get signatures on any two messages of his choice. Show how the adversary can produce a signature on a third message (different from the first two).

Q2: Now suppose a signer needs to sign two message securely. How can he choose a public key and a secret key which will allow him to sign 2 different message (while still keeping security intact).

Q3: Suppose someone gives you an algorithm to solve the CDH problem: it takes as input g, g^a, g^b and output g^ab. Show how you can use this to break Elgamal encryption.

Q4: You work in the IT department of a company. The company employees need to be able to send encrypted emails to each other. However the employees are constantly joining and leaving and the number of employees is large, so its not possible for the each employee to store the public key corresponding to every email ID. Every employee can trust the IT department. Propose a solution for encrypted email communication.
Assume that all adversary is sitting on the network and watching all communication.

HW5: (only for those who know programming) HW: Write a program to compute modular exponentiation: take g (base), e (exponent), and, N (modulas) as input from the user and output the result. Use an online compiler. Do not use the inbuilt functions to compute this. Also, try to make the program as efficient as possible. In particular, please implement the square and multiply algorithm discussed in the class.

Q2: In asymmetric encryption, the client and the server share the key. What if the public key is intercepted and changed on the Internet?

A2: Yeah that’s exactly why we have the notion of certificates. If you use certificates, the certificate will specify the right public key and no adversary can change the certificate

Q4: What’s the meaning of ‘still hard with better than 1/2’ in DDH problem?

A4: It mean even if you guess at random, you will be correct with probability half
Because you are given two things and you have to decide which one is real and which one is random
so certainly with probability half, you can solve it
but solving with probability better than half is hard
